# Project Tips

A concise set of tips to keep development consistent and easy to follow.

## Quick tips

- ✅ Use the VS Code Source Control pane to write clear commit messages.
- ✅ Connect to the Supabase (MCP) server when you need AI-assisted backend work.
- ✅ Create a `copilot-instructions` file at the repo root so AI tools have consistent context.
- ✅ Work with single-file context when focusing on one file to reduce noise.
- ✅ Use `#fetch` when referring to external docs so AI knows which references to follow.
- ✅ Maintain a specific Markdown TODO list:
  - Break tasks into small, reviewable chunks.
  - Commit frequently and check items off as they complete.
  - This keeps the whole application manageable.

## Notes

- Keep instructions short, consistent, and repo-root accessible.
- Store AI context files at the repository root for reliable discovery.
